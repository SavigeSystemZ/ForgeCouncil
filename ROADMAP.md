# Roadmap — Forge Council

Medium-term sequencing. Active tasks live in `PLAN.md`.

## Horizon 1 — Foundations (M0–M1)

- Canonical product docs (`PRD`, `ARCHITECTURE`, `DATA_MODEL`, `NFR`, `RUNBOOK`).  
- `_system/forge-council/` operating extension.  
- Ingestion: `REPO_PROFILE.md`, `CONFLICT_MAP.md`, machine profile alignment.  
- JSON schemas + validation tooling.

## Horizon 2 — Planning and packets (M2–M4)

- Blueprint engine for doc versioning and review loop.  
- Agent profiles, tool/policy bindings, formations.  
- Task packet generation and prompt export pipelines.

## Horizon 3 — Execution and memory (M5–M7)

- Controlled single-milestone runs, artifact ledger.  
- Validation and review gates wired to CI/test adapters.  
- L1–L4 memory operations and resume packet automation.

## Horizon 4 — Scale (M8–M12)

- See `EXTENSION_ROADMAP.md` for multi-runner fabric, comparative eval, release hardening, team governance, plugins.
